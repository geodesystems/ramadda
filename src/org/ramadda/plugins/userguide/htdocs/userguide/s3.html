Section 7.6: AWS S3 Integration
  

7.6.0 Configuring your RAMADDA
RAMADDA uses the AWS CLI. Once you have installed it on your maching define a property in repository.properties in your RAMADDA home directory: 
ramadda.aws.path=/path/to/aws
e.g.:
ramadda.aws.path=/usr/local/bin/aws
For now, RAMADDA uses the --no-sign-request argument to AWS so it does not require a user configuration.
