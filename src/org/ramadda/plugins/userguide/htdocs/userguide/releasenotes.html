Installing and Upgrading

Note: Many users ask if upgrading RAMADDA will effect any existing content. All of the content and files are kept in the RAMADDA home directory and in the database. As long as those directories remain untouched there should not be any problem in upgrading. However, you will need to update any plugins you have installed in the RAMADDA plugins directory.
To update an existing RAMADDA install:
  • Update plugins
    • As noted above there are now 3 plugin collections - core, geo and bio. If you have the old allplugins.jar installed then RAMADDA will update appropriately.
    • If you have individual plugins installed then you should update them to the latest versions.
  • If you are running stand-alone then download and unzip the ramaddaserver.zip from the Geode Systems site.
  • If you are running under Tomcat or Glassfish then download and install the repository.war from the Geode Systems site.

What's new in Version 2.2?

What's new in Version 2.1?

  • The new Slack plugin provides deep integration between RAMADDA and Slack. Browse and search through RAMADDA, create charts and tables on the fly and publish back to Slack.
  • RAMADDA now supports federated search across Google, Wolfram/Alpha, YouTube, Flickr, DuckDuckGo, Arxiv Socrata and other providers. Its easy to build and plug in your own search API.
  • You can integrate Federal Reserve Economic Data - over 200,000 time series of all things economic as well as data from the Energy Information Agency system - over 200,000 time series of all things energy.
  • Some cool dashboard examples.
  • Lots of not so cool bug fixes and UI improvements.

What's new in version 2.0?

  • This is the first release under the new Geode Systems RAMADDA License. You can run the shareware version of RAMADDA, pay a per/user license fee or you can use it for free if you are part of a non-profit organization. Register here.
  • There are now 3 plugin collections - coreplugins, geoplugins and bioplugins. If you have previously installed the allplugins then RAMADDA will automatically delete the allplugins.jar and install the core and geo plugins.
  • The bioplugins provides support for a variety of genomics and medical image data. These services are meant for research and academic purposes only.
  • The RAMADDA Slack plugin provides integration services with the team communciation tool Slack.
  • This release has numerous bug fixes and user interface improvements.
  • Note: The 2.0 release source code is available under SVN revision 5207. 
    svn checkout -r 5207 svn://svn.code.sf.net/p/ramadda/code/ ramadda-code

What's new in version 1.7?

  • Service integration framework - easily incorporate server side command line tools into RAMADDA. RAMADDA repositories will list all available services at: http://geodesystems.com/repository/services/list Examples here. We've integrated a number of tools -
    • Pdfbox - Generate thumbnail images and supports text extraction, split and merge services.
    • Imagemagick - decorate images, strip metadata
    • NCO/CDO - NetCDF Operators and Climate Data Operators tools
    • TEQC - GPS conversion and metadata extraction
    • MB-System - Multi-beam Bathymetry data
  • New services for spreadsheets. RAMADDA now supports displaying spreadsheets (CSV, XLS, XLS) and external databases.
  • Support for a number of Bio-informatics, formats - genomics, imagery, etc.
  • Added support for the Bootstrap CSS framework
  • Lots of new wiki display facilities. Makes it easy to design good looking pages.
  • A number of new entry types - Learning Resource, Case Study, etc.
  • The beginnings of an installer for a standard cloud machine.
  • Note: The 1.7 release is SVN revision r4901 
    svn checkout -r 4901 svn://svn.code.sf.net/p/ramadda/code/ ramadda-code

What's new in version 1.6?

  • This release features a new display front-end integrated into the wiki facility. This enables the creation of rich interactive search, charting and data access interfaces across multiple federated repositories.
  • New entry types - Virtual Group, Case Study, Vocabulary, etc.
  • Extended entry editing including the ability to change entry types.
  • The beginnings of an entry type, metadata and data dictionary editor.
  • Automatic extraction of thumbnail images from powerpoint and word files.
  • And as usual, lots of bug fixes and additions to the documentation.
