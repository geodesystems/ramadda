[ht::head {Configuration Options}] :navleft +callout-info There are lots of ways of configuring your RAMADDA server. Along with these configuration options there are a number of other services that can be configured. See here for more information. -callout :heading Using a CDN By default RAMADDA serves up all of its HTML resources (javascript, css, images etc) from its own server. However, for performance, RAMADDA can be configured to use external content delivery networks (CDN) by setting this property: +pre addCopy=true ramadda.cdnok=true -pre For the core resources RAMADDA uses cdn.jsdeliver.net using the latest Github version tag, e.g.: 
https://cdn.jsdelivr.net/gh/geodesystems/ramadda@6.60.0/...
For other resources RAMADDA uses the standard CDN for the resource, e.g.: 
https://code.jquery.com/jquery-3.3.1.min.js
There are also a number of other flags +pre addCopy=true #Chunk all of the css together ramadda.css.all=true #Serve the javascript minified and chunked ramadda.minified=true #When displaying an entry serve up the beginning of the page before #any processing of the contents of the page ramadda.streamoutput=true #cache the web resources in memory ramadda.cachehtdocs=true -pre :heading Page Templates See templates.html :heading RAMADDA version, contact and security.txt To see the server info go to the ${root}/info link. :p If you want to add a contact for your site then set the property in a .properties file in your RAMADDA home directory: +pre addCopy=true ramadda.contact=... -pre You can add any other information for this page with: +pre addCopy=true ramadda.information=This is my server... -pre RAMADDA also provides a security.txt file that follows the specification from https://securitytxt.org/. The URL is: ${root}/well-known/security.txt.
You can change the contact with the following property: +pre addCopy=true ramadda.security.contact=mailto:youremail@foo.com -pre :heading Archival Resource Keys (ARKS) To configure your RAMADDA to provide [https://arks.org/ Archival Resource Keys (ARKS)] see [arkids.html here]. :heading Multiple Languages RAMADDA supports multiple languages. To enable this set the following property: +pre addCopy=true ramadda.languages.enabled = true ##Set this to define the default language ##ramadda.language.default=es -pre If enabled then logged in users can set their default language. Also there will be a link shown in the upper right of every page to change the displayed language. The translations for each language are defined in a ".pack" file. There are language packs defined in the RAMADDA source (on Github) for Spanish, French and Chinese. You can define your own language packs and place them in the htdocs/languages directory under your RAMADDA home directory. :heading Custom HTTP headers You can specify one or more properties in a .properties file in your RAMADDA home directory. +pre addCopy=true ramadda.httpheader1=<header name>:<header value> ramadda.httpheader2=<header name>:<header value> ... -pre e.g: For specifying HSTS Strict-Transport-Security header you would do: +pre addCopy=true ramadda.httpheader1=Strict-Transport-Security:max-age=86400; includeSubDomains -pre Once you restart your RAMADDA it is a good idea to check the headers, e.g. with curl: +pre addCopy=true curl -v "http://localhost:8080" > foo -pre :heading Embed JSON-LD Metadata JSON-LD is the JSON for Linking Data metadata standard and is used by Google and others to extract structured metadata from a web page. To enable this embedding in entry pages in RAMADDA set the following property in one of the .properties files in your RAMADDA home directory: +pre addCopy=true ramadda.showjsonld=true -pre :heading Tracking Entry access You can enable logging of entry activities by setting the property: +pre addCopy=true ramadda.logging.logactivityfile=true -pre When set various entry activities - view, file, data (providing data to a display) - are logged to a file:
<ramadda home>/logs/entryactivity.log :heading Zoomable Images The Zoomable Image entry type uses an external script, magick-slicer.sh, that is available on Github. This script requires ImageMagic to be installed. Once installed add the below property to your repository.properties file: +pre addCopy=true ramadda.image.slicer=/path/to/magick-slicer.sh -pre :heading Tesseract OCR When ingesting images RAMADDA can extract text from the image using the Tesserarct OCR package. This text can then be used for searching. To configure this first install https://github.com/tesseract-ocr/tesseract. Once installed add the below property to your repository.properties file: +pre addCopy=true ramadda.tesseract=/path/to/tesseract -pre :heading Search Providers RAMADDA has a number of external search providers that are not RAMADDA servers, e.g. Bureau of Labor Statistics, FRED, etc To disable these set: +pre addCopy=true ramadda.search.providers.show=false -pre :heading Using LLM Models See llm.html :heading PurpleAir See purpleair_index.html :heading Generating thumbnails of PDFs Install the Ghostscript package to automatically generate a thumbnail image of the the page of a PDF file by setting the property: +pre addCopy=true service.gs=/path/to/bin/gs -pre :heading Rendering Jupyter Notebooks RAMADDA supports a Jupyter Notebook entry type. It will do its own custom rendering of the notebook. You can also configure RAMADDA to use jupyter to do the rendering by setting the following property value: +pre addCopy=true ramadda.jupyter.path=/path/to/anaconda/bin/jupyter -pre :heading NEON Data You can add NEON data entries but if you want to have a higher rate limit you should obtain an API key from NEON. Go and sign in to NEON. Once signed in create an API key and add it as a RAMADDA property: +pre addCopy=true neon.api.key=YOUR KEY -pre :heading Census data For configuring Census data see /repository/census/index.html :heading FRED, EIA, Quandl, Census data For Federal Reserve, Energy Information Agency, Quandl integration see /repository/biz/index.html :heading Other API keys There are a number of other services that access external APIs and require an API key. :heading Twitter Integration RAMADDA can support the Twitter Card metadata standard. This embeds metadata in an entry page like the below. When a URL to a RAMADDA page is pasted into a post then Twitter will harvest the metadata to provide a display in the post. [ht::pre { }] By default the name of the entry is used for the title. If the entry has a Thumbnail image property attached to it then the Twitter Card will be an image. If the entry is an Image then the image will be used. Else it will just be a text summary. The entry's snippet is used for the description. :p There is also a Twitter Card property type that can be used to specify the title, creator, description and image. Under Add Property->Miscellaneous add the Twitter Card property: [ht::screenshot images/twittercard1.png] Enter any of the values. If defined in the property these will override the defaults of the entry's name, snippet, etc. [ht::screenshot images/twittercard2.png] These properties can be inherited. For example, you can add an inherited Twitter Card property at the top-level of your repository with just the Twitter handle specified. Then for any descendent entry you can specify a title, thumbnail, etc. :p To disable this set the following property in one of the .properties files in your RAMADDA home directory: [ht::pre { ramadda.showtwittercard=false }] :heading Embedding Github checkins RAMADDA supports embedding Github checkins into wiki text with the form: 
@(github user=some_user_name)  
or:
@(github owner=repository_owner repository=repository)
The access to the Github API is rate limited. To increase the request rate create a personal access token through Github and set the following RAMADDA property: +pre addCopy=true github.token=your token -pre [ht::foot]
