[ht::head {Wiki Text} ] <%nochildlist%> The web pages in RAMADDA are created through its wiki facility. The wiki facility supports both regular wiki markup like Wikipedia uses as well as special RAMADDA wiki tags that allow for page payout, displaying maps, lists, charts, etc. [ug::subheading {Basic Markup} basicmarkup] The RAMADDA Wiki facility generally follows the format that Wikipedia follows. For example: [wiki::text { ==Section Heading 1== ===Section Heading 2=== }] A horizontal line: [wiki::text { ---- }] Bulleted lists: [wiki::text { * bullet 1 * bullet 2 ** sub bullet 1 ** sub bullet 2 * bullet 3 }] The RAMADDA wiki text supports a number of other markup methods using +tag/-tag and :tag syntax. The "+/-" syntax is for multi-line and the ":" syntax is for single line. For example, an easy way to add a line break of paragraph (instead of doing <br>, <p>) is: [wiki::text { :br :pr }] To center text do: [wiki::text { +center some text to center -center }] To wrap text in a div do: [wiki::text { +div ... -div }] Set the background color or image with: [wiki::text { +div color="color" ... -div +div image=image_reference ... -div +div style="css style" ... -div }] Where image_reference is either: [wiki::text { +div image=::name_of_attachment -div +div image=this ... -div +div image=other_entry_id ... -div }] To add attributes to the div do: [wiki::text { +div style="max-height:100px;overflow-y:auto;" ... -div }] To create a button do: [wiki::text { :button /url Some label :button-green /url Some label :button-blue /url Some label }] Will produce: :button /url Some label :br :button-green /url Some label :br :button-blue /url Some label :p You can escape your wiki text and include raw, unprocessed text with the <nowiki> tag: [wiki::text { Some wiki text <nowiki> This is text that does not get formatted </nowiki> more wiki text }] For example, if you want to include javascript in your text do: [wiki::text { <nowiki> <script type="text/JavaScript" > ... </script> </nowiki> }] [ug::subsubheading {Macros} {macros}] Macro values can be defined and used later in the wiki page with the :macro and +macro/-macro tags. To define a macro do: [wiki::text { :macro macro_name macro body or: +macro macro_name macro body -macro }] To use the macro do: [wiki::text { :macro macro_name Some text ... any wiki text here ${macro_name} more text }] For example, if you have a number of buttons as described above but want to have one color for them do: [wiki::text { :macro buttonColor -green ... :button${buttonColor} /url1 Button 1 text :button${buttonColor} /url2 Button 2 text }] Repository wide macros can be defined in the server side properties file in the RAMADDA home directory. Add the below properties: [wiki::text { ramadda.wiki.macros=comma separated list of macro names ramadda.wiki.macro.<macro>=<macro value> #For example ramadda.wiki.macros=buttonColor,blockColor ramadda.wiki.macro.buttonColor-green ramadda.wiki.macro.blockColor-olivine }] Note: the block tag below has a similar feature with the "var" attribute. The difference is that the block tag can be placed at the bottom of the page after the var that it references is used. The macro tags need to be inline before they are used. [ug::subsubheading {Blocks} {blocks}] The <block> tag allows you to easily create a toggle block and/or show or hide content based on the user's device. To show a simple toggle block just do: [xml { any content here }] You can specify the toggle label title with: [xml { any content here }] You can specify whether the block is open or closed: [xml { any content here }] You can specify whether to show the content or not: [xml { this will only show for mobile users this will only show for non-mobile users }] You can also use the block tag to define a variable or macro that is used later. [xml { any content here }] This will not show the content in place. Rather you can reference and insert the content elsewhere in the page with: [xml { ${var_name} }] [ug::subheading {Layout and Display Markup} layout] There are a number of layout tags that make it easy to do page layout and display. [ug::subsubheading {Titles and headings} titles] There are a number of block display elements - title, heading, blurb, note, and block that can be used as: [wiki::text { One line forms: :title Some title :heading Some heading :blurb Some blurb :note Some note :block A block Multi line forms: +note Some lengthy text and more -note }] You can change the background color with: [wiki::text { :blurb-green Some green background :note-rose Rose colored background }] Examples at https://ramadda.org/repository/alias/wikiexamples.

[ug::subsubheading {Sections} sections] To make page sections use the +section/-section tags. [wiki::text { +section section content -section }] If you append a "#" then the row number is kept track of and an even/odd color scheme is applied. [wiki::text { +section # ... -section +section # ... -section }] If you append a "----" then a top border is shown: [wiki::text { +section ... -section +section ---- ... -section }] If you add title="..." then a title is added. If you add label="..." then a header is added. The normal use is: [wiki::text { +section # title="section 1" ... -section +section # label="section 2" ... -section +section # label="section 3" ... -section }] Using the {{name}} wiki tag will use the entry name for the title: [wiki::text { +section title={{name}} ... -section }] You can change the background color/image by specifying sections as below. Full list of styles here. [wiki::text { +section class="ramadda-section-map" ... -section +section class="ramadda-section-lightpink" ... -section +section class="ramadda-section-lightblue" ... -section +section class="ramadda-section-lightgreen" ... -section }] You can also add a CSS style descriptor [wiki::text { +section style="background: #888; border: 1px red solid;" ... -section }] [ug::subsubheading {Inset} inset] The +inset tag supports margins. [wiki::text { +inset ... -inset }] This by default gives a 20px left and right margin. To change this you can add any of the attributes, e.g: [wiki::text { +inset top="100" left="50" right="50" bottom="100" ... -inset }] [ug::subsubheading {Frames} frames] Frame your content like this: [ht::cimg images/frame.png {} {border=0}] With the +frame/-frame tags: [wiki::text { +frame content -frame }] You can change the background of the inner part of the frame with: [wiki::text { +frame background=white content -frame }] You can change the size and the color of the frame with: [wiki::text { +frame frameSize=5 frameColor=red content -frame }] Add a shadow effect with [wiki::text { +frame background=#fff frameSize=0 shadow content -frame }] [ug::subsubheading {Embedding Media} embedding] You can embed Twitter, YouTube, Vimeo and other media as: [wiki::text { @(https://twitter.com/Interior/status/463440424141459456) @(https://vimeo.com/515404225k) @(https://www.youtube.com/watch?v=BGaCjM2hal4) @(https://www.facebook.com/don.murray.121/posts/10219628323639292) }] Set dimensions on videos: [wiki::text { @(https://vimeo.com/515404225 width=300 height=200) @(https://www.youtube.com/watch?v=BGaCjM2hal4 width=300 height=200) }] Add a style to the embed: [wiki::text { @(https://vimeo.com/515404225 width=300 height=200 link=true label="Click here" style="display:block;background:red;margin:20px;") }] Add a link to the embed: [wiki::text { @(https://www.youtube.com/watch?v=BGaCjM2hal4 width=300 height=200 start=600 link=true label="Click here") }] Embed Github checkins with: [wiki::text { @(github user=some_user_name) #or a repository @(github owner=repository_owner repository=repository) #don't show avatar image @(github owner=repository_owner repository=repository decorate=false) }] [ug::subsubheading {Embedding Code} codeembed] To embed code do the following: [wiki::text { ```<option language> code here ``` }] Currently only javascript is formatted. For example: [wiki::text { ```javascript let l = []; for(let i=0;<l.length;i++) { console.log(l\[i\]); } var x = {} ``` }] [ug::subsubheading {Embedding Vega Charts} vega] You can embed charts defined in the vega-lite schema directly into a wiki page. More information onv vega-list is available here The format is: [wiki::text { ```vega-lite You vega-lite schema here ``` }] e.g.: [wiki::text { ```vega-lite { "$schema": "https://vega.github.io/schema/vega-lite/v2.json", "description": "A simple bar chart with embedded data. An example vega-lite file.", "width": 360, "height":200, "data": { "values": \[ {"a": "A","b": 28}, {"a": "B","b": 55}, {"a": "C","b": 43}, {"a": "D","b": 91}, {"a": "E","b": 81}, {"a": "F","b": 53}, {"a": "G","b": 19}, {"a": "H","b": 87}, {"a": "I","b": 52} \] }, "mark": "bar", "encoding": { "x": {"field": "a", "type": "ordinal"}, "y": {"field": "b", "type": "quantitative"} } } ``` }] [ug::subsubheading {Grid Boxes} gridboxes] To make a gridded box layout like this use the +gridboxes/+gridbox tags. [ht::cimg images/gridbox.png {} {border=0}] These take the form of: [wiki::text { +gridboxes-<number of columns> +gridbox Header 1 contents -gridbox +gridbox Header 2 contents -gridbox -gridboxes }] For example: [wiki::text { +gridboxes-3 +gridbox Header 1 Hello there
How are you -gridbox +gridbox Hello there
How are you -gridbox +gridbox Header 3 Hello there
How are you -gridbox +gridbox Header 4 Line 1 :br Line 2 -gridbox -gridboxes }] [ug::subsubheading {Scrollable Tables} tables] Scrollable HTML tables are supported with the +table/-table tags. The form is below. Note: any open tr or td tags will be close with the -table tag. [wiki::text { +table height=100 stripe=false hover=true searching=false cellborder=false rowborder=false paging=false ordering=false +tr +td heading 1 -td +td heading 2 -td -tr +tr +td column 1 -td +td column 2 -td -tr ... -table }] The attributes of the +table tag are all optional.

  • height=100 - Set the height of the table and enable scrolling
  • stripe=false - Stripe the rows
  • hover=true - Highlight the rows on hover
  • searching=false - Show the search box
  • cellborder=false - draw borders around the cells
  • rowborder=false - draw borders around the rows
  • paging=false - enable paging
  • ordering=false - enable sorting
There are shortcuts available [wiki::text { +table height=100 :tr "heading 1" "heading 2" +tr :td column 1 :td column 2 -trow ... -table }] [ug::subsubheading {Grid Layout} gridlayout] There is support for the Bootstrap CSS grid system. To do a row layout use the following. The "+col-md-..." correspond to Bootstraps column widths, e.g., col-md-\[2,3,4,6,8,9,10,12\]. Basically you can have any number of columns within a row. Each column has a width, the sum of which adds up to 12. Note: you can also use +col-<some number> as a short hand, dropping the "-md"

[ug::subsubheading {Tabs} tabs] You can create a set of tabs with: [wiki::text { +tabs +tab Tab 1 title Tab 1 content -tab +tab Tab 2 title Tab 2 content -tab ... +tab Tab N title Tab N content -tab -tabs }] Tabs can also be nested: [wiki::text { +tabs +tab Tab 1 title +tabs +tab another tab title ... -tab -tabs -tab +tab Tab 2 title Tab 2 content -tab -tabs }] Use a minimal style: [wiki::text { +tabs min +tab Tab 1 title +tabs -tabs }] Use a minimal style with arrows: [wiki::text { +tabs minarrow +tab Tab 1 title +tabs -tabs }] Center the tabs: [wiki::text { +tabs center +tab Tab 1 title +tabs -tabs }]

[ug::subsubheading {Accordion} accordion] You can create an accordion with the accordian tag. The optional arguments allow you to set the properties of the accordion as described in the Jquery docs. [wiki::text { +accordion heightStyle="auto|fill|content" animate="number of milliseconds" collapsible="true|false" activeSegment="segment number of active segment (0 based)" decorate="true|false" +segment Accordion 1 Title Accordion 1 content -segment +segment Accordion 2 Title Accordion 2 content -segment -accordion }] If decorate=false then no border or background is shown for the headers.
