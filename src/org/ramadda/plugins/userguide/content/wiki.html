[ht::head {Wikis}] <%nochildlist%> RAMADDA uses wiki text in a number of ways. You can create Wiki Page entries, you can style any entry using wiki text and you can also add in a Page Style metadata element that allows you to define the style used for a whole collection of entries. [ug::subheading {Wiki Editor} editor] RAMADDA provides a wiki text editor for use with all of the above ways of using wiki text: [ht::screenshot images/wikiedit.png {Wiki Text Editor} {border=0 width=70%}] The menu bar provides a large number of options for entering the wiki text.
  • Formatting - Basic wiki text formatting - sections, tabs, accoridion, slides, notes, etc.

  • Misc - Further formatting options - title, heading, paragraph, links, etc

  • Entries - Wiki display components used to display RAMADDA entries. For single entries there is name, data, description, download, etc. For multiple entiries there is tabletree, links, map, gallery, etc

  • Import Entry - select an entry then specify how to embed it, e.g. Link, Tree, Gallery, or the entry specifier "entry=ID".

  • Entry ID - Select an entry and insert its ID.

  • Displays -

  • Field ID -

  • Etc... - A live preview window, word count and links to help.
[ug::subheading {Wiki Page Entries} wikipage] One of the entry types that RAMADDA provides is a Wiki Page entry. You create a new Wiki Page just like you create any entry - in a folder that you have create privileges for go to File->New Entry menu. In the form select "Wiki Page" to go to the Wiki Page create form: [ht::screenshot images/wikicreate.png {Wiki Page Create Form}] [ug::subheading {Wiki Text in Entries} wikidesc] You can use the wiki text display capabilities to customize the default HTML view of any other entry or folder types. In the description of the entry click on the checkbox. This enables the wiki text editor: [ht::screenshot images/wikicreate2.png {Wiki Entry Description}] [ht::foot]
