Each row is processed and two Javascript variables are set of each column value - _<column> and _<col#>. For example if you have a table with: x,y 0,1 2,3 4,5 Then each row would have the Javascript variables: 
_x _col0 _y  _col1
So for the Javascript you could do: 
Math.round(_x/_y*100)
You can create multiple columns, e.g.: +pre seesv -func "a,b" "return [1,2]" -pre
